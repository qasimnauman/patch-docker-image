# Privacy Policy

Last updated: 2026-04-22

This Privacy Policy explains how patch-docker-image handles data when you use the public GitHub Action or CLI.

## 1. Scope

This policy applies to the Product repository `qasimnauman/patch-docker-image`, including:

- the GitHub Action defined in `action.yml`;
- the CLI package `docker-vuln-patcher`.

## 2. Data Processed

Depending on how you use the Product, it may process:

- image references (for example, `repo/image:tag`);
- Dockerfile paths, build context paths, and build args provided by you;
- registry credentials you provide via secrets or environment variables;
- vulnerability scan artifacts generated by Docker Scout (for example JSON reports and patch plans);
- workflow/job logs generated by your CI platform.

## 3. How Data Is Used

Data is used only to perform patching and verification tasks, including:

- pulling or building source images;
- scanning for vulnerabilities;
- generating patch plans and patched Dockerfiles;
- building, verifying, and optionally pushing patched images.

## 4. Data Storage and Retention

- The Product does not run a hosted backend operated by Publisher.
- Data is processed and stored in your own execution environment (for example your GitHub runner or local machine).
- Reports are written to the configured report directory (default: `./vuln_reports`).
- Retention is controlled by you. You are responsible for deleting artifacts according to your policy.

## 5. Data Sharing

The Product may send data to third-party services that you choose to use, such as:

- GitHub Actions (workflow execution);
- container registries (image pull/push);
- Docker Scout (vulnerability analysis).

Publisher does not sell your personal data.

## 6. Credentials and Security

- Credentials are expected via environment variables or secret managers.
- Registry authentication uses secure stdin patterns where supported (`docker login --password-stdin`).
- You are responsible for secret configuration, access controls, and least-privilege token usage in your environment.

## 7. Your Responsibilities

You are responsible for:

- ensuring you have a lawful basis to process any data used with the Product;
- providing notices/consents required by applicable law;
- configuring artifact retention and access controls in your systems.

## 8. International Transfers

Because execution occurs in your chosen environment and providers, international transfers depend on your infrastructure and vendor choices.

## 9. Changes to This Policy

This policy may be updated from time to time. The current version in the default branch is the effective version.

## 10. Contact

For privacy-related questions, open an issue in this repository:

[GitHub Issues](https://github.com/qasimnauman/patch-docker-image/issues)
